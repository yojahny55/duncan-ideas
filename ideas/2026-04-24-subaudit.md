**Date:** 2026-04-24

**Inspiration:** Subscription fatigue trending on Twitter/X — 41% of consumers feel overwhelmed, average company uses 130 apps, people rotating streaming services monthly.

---

## Problem Statement

People and small businesses are drowning in subscriptions. The average person manages 12+ recurring payments; the average small business, 130 SaaS tools. Nobody knows what they're actually paying for, what's duplicating functionality, or what they haven't used in months. Canceling is deliberately painful. The result: billions wasted annually on zombie subscriptions.

Existing tools (Rocket Money, Truebill) focus on bank transaction parsing — reactive, not strategic. They tell you *what you paid*, not *what you should pay for*.

## Solution

**SubAudit** is a subscription intelligence platform that goes beyond tracking:

1. **Auto-discovery** — Scans email receipts, bank transactions, and browser autofill to build a complete subscription map
2. **Usage tracking** — Monitors actual app/tool usage via lightweight integrations (OAuth, browser extension, mobile app screen time)
3. **Overlap detection** — AI identifies duplicate functionality across subscriptions (e.g., 3 project management tools, 2 note-taking apps)
4. **Optimization engine** — Suggests plan downgrades, annual switches, free alternatives, and bundling opportunities
5. **One-click cancel** — Automated cancellation flows with card-locked virtual cards for stubborn vendors
6. **Team audit** — For small businesses: see who's using what, eliminate seat waste

### Killer Feature: "Subscription Score"
A single number (0-100) representing how efficiently you're spending on subscriptions. Track it over time. Share it. Gamify saving money.

## Tech Stack

### Frontend
- **Next.js 15** with App Router + React Server Components
- **Tailwind CSS v4** + shadcn/ui components
- **Recharts** for spending visualizations
- **Framer Motion** for micro-interactions

### Backend
- **Node.js** with Hono (lightweight, fast)
- **PostgreSQL** (Supabase) — subscription data, user accounts
- **Qdrant** — vector similarity for overlap detection
- **Plaid API** — bank transaction ingestion
- **Gmail API** — receipt scanning

### AI / Intelligence
- **Gemini 2.5 Flash** — categorization, overlap analysis, recommendations
- **Embeddings** (gemini-embedding-001) — similarity search between subscription descriptions

### Infrastructure
- **Vercel** — frontend hosting
- **Railway** — backend + Postgres
- **Browser extension** — Chrome + Firefox (Manifest V3) for usage tracking
