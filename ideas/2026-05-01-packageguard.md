## Problem Statement

AI coding agents (Claude Code, Cursor, Copilot, etc.) routinely suggest packages that **don't exist** or reference **malicious typosquatting libraries**. Developers copy-paste or auto-accept these suggestions, leading to:

1. **Build failures** — `npm install` or `pip install` fails mid-deployment
2. **Security vulnerabilities** — typosquatted packages injected by attackers exploiting AI hallucination patterns
3. **Wasted debugging time** — engineers spend hours chasing phantom dependency errors
4. **Supply chain risk** — AI agents auto-generating `package.json` / `requirements.txt` with fake entries that get committed to production

The core issue: **there's no real-time validation layer between AI output and your dependency tree.**

## Solution

**PackageGuard** — a CLI tool + VS Code extension + GitHub Action that intercepts AI-suggested dependencies and validates them before they reach your codebase.

### How It Works
1. **Scan** — Monitors AI agent output (Claude Code, Cursor, Copilot) for `import`, `require`, `use`, and package manager commands
2. **Validate** — Checks each dependency against real registries (npm, PyPI, crates.io, Maven, Go modules)
3. **Score** — Assigns a trust score based on: package age, download count, maintainer reputation, known vulnerabilities (OSV/Advisory DB)
4. **Alert** — Blocks or warns on: non-existent packages, low-trust packages, typosquatting patterns, known CVEs
5. **Suggest** — Recommends the real package the AI *probably* meant

### Key Features
- **Real-time scanning** — hooks into AI agent stdout/stderr
- **IDE integration** — squiggly lines on hallucinated imports
- **CI/CD gate** — GitHub Action fails PRs with unverified dependencies
- **Historical audit** — scan existing `package.json` / `requirements.txt` for AI-introduced risks
- **Team dashboard** — track which AI agents hallucinate most, common fake packages across org
