# SubFeed — Kill Subscription Creep

**Date:** 2026-06-03
**Source Inspiration:** Reddit r/ProductivityApps, r/personalfinance — recurring complaints about "subscription creep," forgotten recurring charges, and inability to manage all subs in one place. 41% of users (Bango 2025) can't manage subscriptions centrally. 62% want bundling over individual signups.

---

## Problem Statement

People are drowning in subscriptions. The average American has **12+ active subscriptions** and underestimates monthly spend by **$100-200**. The problems:

1. **Forgotten subscriptions** — signed up for a free trial, never canceled, now paying $15/mo for something unused
2. **No unified view** — subscriptions scattered across email, credit card statements, app stores, and bank accounts
3. **Cancel friction** — companies deliberately make cancellation hard (dark patterns, phone-only cancel, retention loops)
4. **No spend awareness** — no "shock moment" showing total monthly/yearly burn
5. **Duplicate services** — paying for Spotify AND Apple Music, Netflix AND Hulu, without realizing overlap

Reddit threads consistently rage about: "I just found out I've been paying for X for 8 months and never used it."

## Solution: SubFeed

A smart subscription tracker and optimizer that:

1. **Auto-discovers subscriptions** by scanning email receipts (Gmail/Outlook OAuth), bank transactions (Plaid), and app store purchases
2. **Dashboard with "Shock Number"** — your real monthly/yearly subscription burn rate
3. **Usage detection** — optional integration to see which subscriptions you actually use vs. pay-for-nothing
4. **One-tap cancel assistant** — provides direct cancel links, pre-fills cancellation emails, flags dark patterns
5. **Smart suggestions** — identifies duplicates, cheaper alternatives, and annual vs monthly savings
6. **Trial tracker** — alerts 3 days before free trials convert to paid
7. **Family mode** — see combined household subscription spend, find shared-plan eligible services

### Key Differentiator
Unlike existing tools (Rocket Money, Truebill/Subscriptions), SubFeed is **open-core** with a privacy-first approach — data stays on-device or in your own encrypted vault. No selling anonymized data. No requiring bank connections if you prefer manual/email-only scanning.

## Tech Stack

| Layer | Choice | Why |
|-------|--------|-----|
| **Frontend** | Next.js 15 + React 19 + Tailwind CSS | Fast, SSR, great DX |
| **Mobile** | React Native (Expo) | Shared codebase with web |
| **Backend** | Node.js + Hono | Lightweight, fast, edge-deployable |
| **Database** | SQLite (local) + Turso (sync) | Privacy-first, user owns data |
| **Auth** | Clerk | Quick setup, passkey support |
| **Email scanning** | Gmail API + Nodemailer parser | Auto-detect subscription receipts |
| **Bank linking** | Plaid (optional) | Transaction-based sub discovery |
| **AI** | Local embeddings (Transformers.js) | Categorize subscriptions on-device |
| **Payments** | Stripe | For premium features |
| **Hosting** | Vercel (web) + Expo EAS (mobile) | Zero-config deployment |
